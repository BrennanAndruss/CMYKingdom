CMYKingdom

Brennan Andruss, Camila Yeremin, Hahns Pena, Joaquin Arredondo, Lori Gavia

Description

You find yourself in a magical kingdom void of color. Your quest is to explore the world to collect cyan, magenta, and yellow crystals scattered throughout the land to restore color to the kingdom!

CMYKingdom was made over 10 weeks with C++ and OpenGL for CSC 476: Real-time 3D Computer Graphics Software.

    • Technologies

    Scene Organization

    We implemented a component-based object system for scene organization, with hierarchy handled through Transform classes owned by each object. Hierarchy is implemented in the Transform class to handle local and world space transformations. The hierarchy enables grouping of objects to support shadow mapping and speed up view frustum culling.

    The engine has various components for key object functionality, and additional components can be created and added to function as "script" components. Objects and components are created in the game-side code and added to the scene.

    The scene class holds references to various scene components, including cameras and lights. The main camera can be switched during runtime, used to switch between gameplay and editor cameras. The list of lights and objects are fetched by the renderer for render passes.

    Complex Environment

    The terrain was scuplted and textured using Unity's terrain tool. The heightmaps and splatmaps are exported and loaded into our engine, where the heightmap is used to generate a mesh and heightmap collider and the splatmaps are used for the terrain shader.

    Additional game-side shaders for the environment included the water shader and instanced-rendered grass shader.

    Physics and Collisions

    To handle collision detection efficiently with a heightmap terrain and lots of assets, we integrated the Bullet Physics library. Setup is handled in the engine's physics system, and components for rigidbodies and various colliders are implemented to sync the Bullet physics world with the engine-side object transforms for accurate rendering.

    Skeletal Animation

    Our player character has an array of animations, driven by skinned skeletal animation and blending between animations. Animation and other asset loading is handled with Assimp, and our character model and animations were hand-made!

    Render Pipeline

    We implemented a modular render pipeline extensible with various render passes. Rendering techniques used include deferred rendering, HDR rendering, skybox rendering, and screen-space refractions for water rendering.

    Cascade Shadow Mapping

    To support a large complex environment, cascade shadow mapping is implemented. Percentage closer filtering with poisson disk sampling and shadow samplers are used to create soft shadows.

    Level Editor

    To build out our world, we implemented an in-game level editor to handle editing and creating objects and their components. The editor enables saving and loading of scenes, allowing for a customizable experience traveling through the world.
